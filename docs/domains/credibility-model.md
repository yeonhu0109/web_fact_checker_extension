# 신뢰도 모델 (Credibility Model)

## 판정 레이블 정의

| 레이블 | 점수 범위 | 의미 | 색상 |
|--------|----------|------|------|
| `true` | 0.8 ~ 1.0 | 사실임 — 검증 가능한 출처가 확인됨 | 초록 (#4caf50) |
| `mostly_true` | 0.6 ~ 0.8 | 대체로 사실 — 일부 맥락/세부사항이 누락되었으나 핵심은 사실 | 연초록 (#8bc34a) |
| `unverified` | 0.4 ~ 0.6 | 확인 불가 — 검증할 수 있는 출처를 찾을 수 없음 | 노랑 (#ffeb3b) |
| `mostly_false` | 0.2 ~ 0.4 | 대체로 허위 — 일부 사실적 요소가 있으나 전반적으로 오해의 소지 | 주황 (#ff9800) |
| `false` | 0.0 ~ 0.2 | 허위 — 사실과 다르거나 조작된 정보 | 빨강 (#f44336) |
| `opinion` | - | 의견/주장 — 사실이 아닌 개인적 견해, 판정 대상 아님 | 회색 (#9e9e9e) |

## 점수 산정 방식

### 개별 문장 점수 (Statement Score)
- LLM이 자체 추론을 통해 0.0~1.0 범위의 점수를 직접 할당
- 점수 산정 근거 (LLM 프롬프트에 포함된 지침):
  - **출처 명시 여부**: 주장에 구체적인 출처가 인용되었는가
  - **검증 가능성**: 주장이 객관적으로 검증 가능한 사실인가
  - **일관성**: 널리 알려진 사실과 일치하는가
  - **구체성**: 모호하지 않고 구체적인 내용인가
  - **의견 여부**: 사실(fact)인가 의견(opinion)인가

### 전체 페이지 점수 (Overall Score)
```
overallScore = (non-opinion statements의 score 평균)
```

- `opinion` 레이블 문장은 집계에서 제외
- 전체 문장이 모두 `opinion`일 경우: `overallScore = null`, "본문이 주로 의견/주장으로 구성됨" 메시지 표시

## LLM 프롬프트 설계

### 시스템 프롬프트
```
당신은 웹 페이지의 사실 여부를 검증하는 팩트체크 AI입니다.
주어진 텍스트의 각 문장을 분석하여 사실(fact)과 의견(opinion)을 구분하고,
사실인 문장에 대해 신뢰도 점수와 판정 레이블을 할당하세요.

판정 레이블: true, mostly_true, unverified, mostly_false, false, opinion
- opinion은 사실 판정 대상이 아닌 개인적 견해나 추측에 사용하세요.

JSON 배열 형식으로만 응답하세요. 추가 설명이나 마크다운 없이 순수 JSON만 반환하세요.
```

### 분석 프롬프트 템플릿
```
다음 웹 페이지의 내용을 분석해주세요.

--- 분석할 텍스트 시작 ---
{extractedText}
--- 분석할 텍스트 끝 ---

각 문장(또는 논리적 단위)에 대해 다음 JSON 배열을 반환하세요:
[
  {
    "text": "분석할 문장/단위",
    "score": 0.0에서 1.0 사이의 점수,
    "label": "true | mostly_true | unverified | mostly_false | false | opinion",
    "explanation": "한국어로 1-2문장 근거 설명"
  }
]

점수 기준:
- 0.8-1.0: 명확히 사실, 검증 가능한 출처 존재
- 0.6-0.8: 대체로 사실, 일부 맥락 부족
- 0.4-0.6: 확인 불가, 검증 어려움
- 0.2-0.4: 대체로 허위, 오해의 소지
- 0.0-0.2: 명확히 허위
- opinion: 사실 판정 대상이 아닌 의견/주장 (score와 explanation 생략 가능)
```
