# 도메인 문서 (Domain Documents)

## 목적

이 디렉토리는 Web Fact Checker 확장 프로그램의 **도메인 지식**을 문서화합니다.
코드가 구현하는 비즈니스 로직, 판정 기준, 분석 흐름을 코드와 분리하여
단일 진실 공급원(Single Source of Truth)으로 관리합니다.

## 도메인 경계 맵

```
┌─────────────────────────────────────────────────┐
│                  팩트체크 시스템                    │
│                                                   │
│  ┌──────────────┐   ┌──────────────────┐         │
│  │  도메인 모델   │   │  분석 파이프라인    │         │
│  │  (domain-    │   │  (analysis-      │         │
│  │   model.md)  │──▶│   pipeline.md)   │         │
│  └──────────────┘   └────────┬─────────┘         │
│                              │                    │
│                              ▼                    │
│  ┌──────────────────────────────────┐            │
│  │      신뢰도 모델                  │            │
│  │      (credibility-model.md)      │            │
│  └──────────────────────────────────┘            │
└─────────────────────────────────────────────────┘
```

## 문서 목록

| 문서 | 설명 |
|------|------|
| `domain-model.md` | 도메인 모델 정의, 유비쿼터스 언어, 엔티티/값 객체, 도메인 이벤트 |
| `analysis-pipeline.md` | 분석 흐름 상세, LLM 호출 전/후처리, 재시도·에지 케이스 |
| `credibility-model.md` | 신뢰도 판정 기준, 점수 산정 공식, 레이블 정의, LLM 프롬프트 설계 |

## 변경 규칙

- 도메인 문서는 코드 리팩토링보다 **우선**하여 갱신합니다.
- 도메인 모델의 변경은 반드시 이 문서들에 반영한 후 코드를 수정합니다.
- 새로운 LLM Provider를 추가할 때 `domain-model.md`의 Provider 정의를 추가합니다.
